#### 오버로딩(Overloading)

간단하면서도 다형성을 지원하는 방법이 **오버로딩(overloading)** 이다.

프로그래밍에서 **오버로딩** 이란 메소드 함수명은 같지만 전달받는 인자 값에 의해 호출되는 메소드가 다르게 하는 것이다.



예시 ( **overloading** 을 사용하지 않았을 때)

```java
sumTwo() 호출       // x+y 를 할 때 

public static int sumTwo(int x, int y) {
	return x+y;
}


sumThree() 호출     // x+y+z 를 할 때
    
public static int sumThree(int x, int y, int z) {
    return x+y+z;
}
```

위 코드는 **오버로딩(overloading)** 을 사용하지 않을 때의 문제점을 보여준다.

즉, 가독성이 떨어지고 귀찮아진다. 그래서 오버로딩을 사용하면 위의 코드를 아래처럼 바꿀 수 있다.


예시 ( **overloading** 을 사용했을 때 )

```java
sum() 호출

public static int sum(int x, int y){
	return x+y;
}

public static int sum(int x, int y, int z) {
	return x+y+z;	
}
```

오버로딩을 사용하면 메소드 명이 같아도 전달되는 매개변수의 개수에 따라 정의된 메소드가 호출된다.

**즉 오버로딩은 메소드 이름, 반환 타입과는 상관없이 메소드 이름이 같으면 매개변수의 타입과 개수에 의해 **
**호출되는 메소드가 결정된다.**



#### 오버로딩을 하기 위한 조건

1. 오버로딩은 메소드 이름이 같아야 한다.
2. 매개변수의 개수 또는 타입이 달라야 한다. 당연히 매개변수로 전달되는 인자의 순서도 같아야 한다.
3. 매개변수는 같고 반환 타입이 다른 경우는 오버로딩이 성립되지 않습니다.



#### 오버로딩의 장점

1. 가독성이 증가한다.
2. 하나의 이름만 기억하면 되므로 오류의 가능성을 많이 줄일 수 있다.
3. 메소드의 이름만 보고, "이름이 같으니 같은 기능이겠구나" 하고 유추할 수 있다.
4. 메소드 이름을 절약할 수 있다.





#### 오버라이드 (Override)

프로그래밍에서는 오버라이드를 덮어씌우는 것으로 생각하면 좋다.
상속 관계인 부모 클래스와 자식 클래스 사이에서 부모 클래스의 메소드를 똑같이 가져와 사용하는 것이다.

예시를 통해 알아보쟈.

```java
String tell(){
	return "Terran Class";
}

String tell(){
	return "Barrack Class";
}

String tell(){
	return "Marine Class";
}
```

상속 관계에서는 부모 클래스의 리소스를 자식 클래스가 사용할 수 있다 했는데 같은 리턴 타입, 메소드명, 매개변수를 가지고 재정의하여 사용했다.

이것이 바로 오버라이드라는 것인데 부모 클래스가 정의한 함수를 덮어씌워 다시 정의하여 사용하는 것이다.
하지만 여기서 오버라이드를 한다고 부모 클래스의 기능이 사라지는 것은 아니다.



```java
class Example {
    public static void main(String[] args) {
        Terran b = new Marine();
        System.out.println(b.tell());
        System.out.println(b.getResource());
    }
}
```

다음과 같이 Marine 객체를 생성하여 **tell()** 메소드를 호출하면 **Marine** 클래스에 정의된 메소드가 호출한다.
즉 Barrack 객체를 생성하여 **tell()** 메소드를 호출한다면 **Barrack** 클래스에 정의된 메소드가 호출된다는 것이다.



이런 식으로 부모 클래스의 메소드를 사용할 수 있어도 자식 클래스에서 변경해야 할 상황이 발생한다면 오버라이드를 통해 자식 클래스에서만 새로운 기능으로 재정의할 수 있습니다.



#### 오버라이드를 하기 위한 조건

1. 부모 클래스와 자식 클래스 사이에서만 성립된다.

2. static 메소드는 클래스에 속하는 메소드이기 때문에 상속되지 않고 오버라이드 되지도 않는다.

   **: 이 부분은 static이 객체가 생성될 때가 아니라 클래스가 생성되고 한번만 메모리에 생성된다는 개념을 숙지하였다면 금방 이해가 갈 것이다.**

3.  private의 접근제어자를 가진 메소드는 상속 자체가 되지 않아 오버라이드도 성립되지 않는다.
4. interface를 구현하여 오버라이드할 때는 반드시 public 접근 제어자를 사용해야 한다.
5. 오버로드와 달리 리턴 타입, 메소드 명, 매개변수 패턴이 모두 같아야 한다.

6. 부모 클래스의 메소드의 접근 제한자 범위보다 작아질 수 없고 확장은 가능하다.
7. 부모 클래스의 메소드보다 더 많은 예외를 던질 수 없다.

8.  final이 지정된 메소드는 오버라이드할 수 없다.





#### 오버로딩의 장점

오버라이드는 메소드 하나로 여러 객체를 다루고 객체마다 다른 기능을 사용할 수 있다.
